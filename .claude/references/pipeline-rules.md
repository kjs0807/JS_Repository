# 파이프라인 공통 규칙

이 문서는 strategy-builder, strategy-developer, strategy-full, build-binary 스킬이 공유하는 자동 진행 규칙을 정의한다.

---

## 자동 진행 규칙

### 절대 멈추지 마라

파이프라인 실행 중 **각 스킬에 정의된 "멈출 수 있는 지점" 외에서는 절대 멈추지 마라.**

```
금지 행동:
- Agent 결과를 받은 후 요약만 출력하고 다음 단계를 안 하기
- "결과를 확인하겠습니다" 하고 멈추기
- "다음 단계로 진행할까요?" 같은 불필요한 확인 요청
- 중간 산출물을 길게 출력하며 시간 낭비
- 사용자 질문에 자유형 대화로 전환하기

필수 행동:
- Agent 완료 → 결과 파일 저장 → 1줄 상태 업데이트 → 즉시 다음 단계 진입
- 예: "Phase 2 완료 (data_plan + research_brief + risk_plan 저장). Phase 3 진입."
```

### 파이프라인 잠금 메커니즘

파이프라인 시작 시 `PIPELINE_LOCKED = true` 상태에 진입한다.

```
잠금 상태 규칙:
1. 사용자가 질문하면: 1줄 답변 후 즉시 다음 단계 속행
   예: 사용자 "이 전략이 잘 될까?" → "검증 단계에서 확인합니다. 현재 Phase 4 진행 중."
2. 사용자가 설정 수정을 요청하면: 해당 필드만 수정 후 즉시 속행
3. 사용자가 "멈춰" / "중단" 하면: 체크포인트 저장 후 중단
4. 그 외 모든 경우: 파이프라인 속행 (자유형 대화 전환 금지)

잠금 해제 조건:
- 파이프라인 정상 완료
- FAIL 판정 (실패 종료)
- 사용자 명시적 중단 요청
```

### 멈췄다 재개 시 복구

사용자가 "계속해", "이어서" 등을 입력하면:
1. 체크포인트 파일 확인 → 현재 완료된 단계 파악
2. 이전 단계의 산출물 파일 존재 확인
3. 다음 pending 단계부터 이어서 실행
4. **이미 완료된 단계를 다시 실행하지 마라**

### 에이전트 실패 시 복구

```
에이전트가 실패(타임아웃, 에러)한 경우:
1. 에러 내용을 1줄로 기록
2. 동일 에이전트를 1회 재시도 (프롬프트 동일)
3. 재시도도 실패 → Lead가 해당 에이전트의 핵심 작업만 직접 수행
4. 직접 수행도 불가 → 사용자에게 보고 후 판단 대기
5. 최대 재시도 횟수: 2회 (원본 1회 + 재시도 1회)
```

### 비용 절감

- 모든 에이전트는 `model="sonnet"` 사용
- Lead만 Opus
- 단계 간 결과를 파일로 전달하여 컨텍스트 효율화
